# kuaimai-cli 每阶段新增能力

本文档记录各开发阶段完成后系统**新增**的能力，便于版本对照与验收。  
**当前代码版本**：`meta_data.json` **v1.2.0**，业务 shortcuts 对接 **erp-items-core**（`https://erp1.superboss.cc/`），路径**不**追加 `.json` 后缀。

---

## 当前业务里程碑：商品标题能力（已基本可用）

通过 CLI 可完成与页面一致的**商品标题**相关操作：

| 步骤 | 命令 | 说明 |
|------|------|------|
| 1. 按标题查商品 | `item +list` / `item list` | `--body` 中 `title` 模糊/精确筛选（form → `/item/stock/queryList`） |
| 2. 取系统 ID | 列表 `data` 中 `sysItemId` | 可与 `count` 核对总数 |
| 3. 拉详情（可选） | `item get-detail --sys-item-id <id>` | GET `/item/getItemDetail` |
| 4. 改标题保存 | `item update-title`（推荐）或 `item save` | `update-title` 内部 get-detail 合并后 POST `/item/saveItem`；支持 `--dry-run` |

鉴权：`auth login` 后请求头自动携带 `accessToken`（见 [本地使用实操指南](./kuaimai-cli%20本地使用实操指南.md)）。

> **历史说明**：早期迭代曾对接 erp-scm（scm3、9 业务域、`.json` 后缀）。当前仓库已收敛为 **erp-items-core 商品域**，下文阶段二中的「多环境 / scm3」等条目仅作架构演进记录，**以当前 `shortcuts/item` 与 v1.2.0 元数据为准**。

## 阶段一：基建骨架版（已完成）

**定位**：最小生产可用（MVP），约 **60%** 飞书体验。  
**目标**：配置、鉴权、原始 API、结构化输出、业务 shortcuts 形态（飞书 `+` 语法）。

### 1. 架构与工程

| 能力 | 说明 |
|------|------|
| 分层目录 | `cmd/` · `internal/` · `shortcuts/` · `pkg/` · `skills/` |
| 单二进制 | `make build` / `go build -o kuaimai-cli` |
| 分层铁规 | 业务仅在 `shortcuts/`；`cmd` 无网络/业务逻辑 |

### 2. 全局能力

`--help` · `--verbose` · `--output table|json` · `--dry-run`

### 3. 配置管理

| 命令 | 说明 |
|------|------|
| `config init` | 创建 `~/.kuaimai-cli/config.yaml` |
| `config get [key]` | 查看配置 |
| `config set <key> <value>` | 修改并落盘 |

**调用链**：`config init` → `internal/config.Init()` → `template.go` → `~/.kuaimai-cli/config.yaml`（0700/0600）

**输出优先级**：`--output` > `cli.output` > `table`

**默认 API**：`https://erp1.superboss.cc/`

### 4. 鉴权

| 命令 | 说明 |
|------|------|
| `auth login <accessToken>` | 写入系统密钥链（**唯一方式**）；可选 `--profile`、`--api-url` |
| `auth status` / `auth logout` / `auth check` | 状态 / 清除 / 探测 token 与 API |
| `auth list` / `auth use <profile>` | 多账号 profile 列表与切换 |

请求头：`accessToken: <token>`（非 Bearer；由 `auth login` 写入密钥链后自动附加）

### 5. 原始 API

```bash
kuaimai-cli api GET|POST|PUT|DELETE <path> [--body '{}']
```

### 6. 输出规范

stdout：`{ok, data, error, hint}` · stderr：日志 · 无堆栈

### 7. 阶段一尚未具备（由阶段二补齐）

`schema`/`service`、彩色输出、completion、重试、`--page-all`、form 列表等 → 见阶段二。

## 阶段二：标准企业版（已完成）

**定位**：正式企业版基建，约 **80%** 飞书体验（执行管线 + 元数据命令 + 终端体验）。

### 1. 统一执行管线（shortcuts/common）

| 组件 | 说明 |
|------|------|
| `runner` | `ExecuteList` / `ExecuteWrite`、鉴权、dry-run |
| `BuildPath` / `SetQuery` | GET 查询参数拼接 |
| `RunGET` / `RunPOST` / `RunPOSTForm` / `RunPOSTFormListMap` | 各 HTTP 形态封装 |

### 2. 元数据驱动（registry + service + schema）

| 组件 | 路径 | 说明 |
|------|------|------|
| 注册表 | `internal/registry/meta_data.json` | 当前 **v1.2.0**，1 service / 4 operations（item 域） |
| 命令 | `kuaimai-cli service item <op>` | 如 `service item list --body '{}'` |
| 自省 | `kuaimai-cli schema` | 输出全部 operation |
| 拉取 | `scripts/fetch_meta/fetch_meta.sh` | 可选 `KUAIMAI_OPENAPI_URL` |

### 3. 鉴权（accessToken）

`auth login <accessToken>` → 密钥链 → 请求头 `accessToken`

### 4. 终端体验

彩色 table · `cli.color` · `--no-color` · `completion bash|zsh|powershell`

### 5. 网络层

`api.retry`（默认 3）· 指数退避 · `--page-all`（识别 form/JSON 中 `pageNo`/`pageSize`）

### 6. dry-run

`service` 写操作 · `api POST|PUT|DELETE` · `item save` 等写 shortcuts

### 7. 阶段二验收命令速查

```bash
kuaimai-cli config init
kuaimai-cli auth login "<accessToken>"
kuaimai-cli schema --output json
kuaimai-cli service item list --help
kuaimai-cli api POST /item/stock/queryList --body '{}' --verbose
```

### 8. 阶段二尚未具备

erp-items-core 业务 shortcuts 闭环 · Skill · csv/ndjson · 连接池/熔断 · 脱敏 · 审计 → 见阶段三。

## 阶段三：平台进阶版（已完成，erp-items-core 商品域）

**定位**：平台进阶 ~95%，围绕 **商品标题查改** 与 Agent 协作补齐能力。

### 1. erp-items-core 商品 shortcuts

| 子命令 | Method | 路径 | 说明 |
|--------|--------|------|------|
| `list` / `+list` | POST form | `/item/stock/queryList` | 库存列表；`title` 等筛选与浏览器 form 一致 |
| `count` | POST form | `/item/stock/queryCount` | 与 list 相同筛选条件下的总数 |
| `get-detail` | GET | `/item/getItemDetail` | `--sys-item-id` |
| `save` | POST JSON | `/item/saveItem` | 新建/修改；复杂字段改写字段 |
| `update-title` | GET+POST | `getItemDetail` + `saveItem` | **改标题**推荐；`--sys-item-id` + `--title` |

**带 `+` 的快捷命令**：`item +list`。

**常用 flags**：

- `list`/`count`：`--body` JSON（转为 `application/x-www-form-urlencoded`）
- `get-detail`：`--sys-item-id`；可选 `--api-name`
- `save`：`--body`（SysItemModel JSON）；`--sync-hs-code`；`--dry-run`
- 全局：`--page-all`、`--output csv|ndjson`

**列表默认值**（与 ARCHIVE_V2 商品库存页对齐）：`pageType=ITEM_STOCK`、`subPageType=ARCHIVE_V2`、`api_name=item_stock_queryList` 等，见 `shortcuts/item/item.go`。

### 2. Skill 平台

| 命令 | 说明 |
|------|------|
| `skill list` | 扫描 `./skills` 与 `~/.agents/skills` |
| `skill install <name>` | 安装单个 Skill 到 `~/.agents/skills/` |

内置 Skill：`skills/kuaimai-item/SKILL.md`、`skills/kuaimai-shared/SKILL.md`。

### 3. 多格式输出

| `--output` | 行为 |
|------------|------|
| `csv` | 列表成功时输出 CSV |
| `ndjson` | 列表成功时每行一条 JSON |
| `json` / `table` | 保持 `{ok,data,error,hint}` 信封 |

### 4. 企业级网络

| 配置项 | 说明 |
|--------|------|
| `api.pool_max_idle` | 连接池最大空闲连接（默认 100） |
| `api.pool_max_idle_per_host` | 每主机空闲连接（默认 10） |
| `api.circuit_threshold` | 连续失败触发熔断阈值（默认 5） |
| `api.circuit_cooldown_sec` | 熔断冷却秒数（默认 30） |

### 5. 脱敏与安全

- `pkg/sanitize`：日志 / dry-run 中 `accessToken`、`password` 等脱敏
- Token 仅通过 `auth login`，请求自动带 `accessToken` 头

### 6. item 增强

| 能力 | 说明 |
|------|------|
| `--page-all` | `item list`/`+list` 按 form 中 `pageNo`/`pageSize` 自动翻页 |
| 审计日志 | `~/.kuaimai-cli/audit.log` 记录业务命令 |

### 7. Agent 约定

根目录 `AGENTS.md` 已约定 item 域优先 shortcuts、输出格式与安全（阶段四扩展 E2E / 升级）。

### 8. 阶段三验收命令速查（标题能力）

```bash
kuaimai-cli auth login "<accessToken>"

# 按标题查
kuaimai-cli item +list \
  --body '{"title":"关键字","pageNo":1,"pageSize":20}' \
  --output json

# 改标题（推荐 update-title，先试跑加 --dry-run）
kuaimai-cli item update-title \
  --sys-item-id <id> \
  --title "新标题" \
  --dry-run --verbose

kuaimai-cli doctor --output json
kuaimai-cli skill list
kuaimai-cli skill install-all --from ./skills
```
